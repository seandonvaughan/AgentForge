- **Recommended Tests:**
  - `computeMemoryStats excludes failed and running cycles from denominator`
  - `hit rate correctly uses audit.json memoriesInjected field when present`

- **Changes:** Approval SSE handler now checks both `approval.pending` (dot) and `approval_pending` (underscore) for backward compatibility. Added `_modalShownFor` Set to prevent duplicate modal shows. Modal only opens if no overlay currently visible.

- **Risk Factors:**
  - **Race condition:** Modal visibility check (`overlay.classList.contains('visible')`) may not catch all in-progress states if async show/hide not fully synchronous.
  - **Session-local state:** `_modalShownFor` Set is not persisted; page reload clears it (minor usability issue, not a bug).
  - **Backward compat:** Dual notation (dot + underscore) adds subtle branch coverage gap — only one path exercised at a time in production.
- **Recommended Tests:**
  - Playwright e2e: SSE event arrives → modal opens (happy path)
  - Playwright e2e: second event for same cycleId arrives → modal does NOT re-open
  - Playwright e2e: overlay visible, new event arrives → no second modal attempt
