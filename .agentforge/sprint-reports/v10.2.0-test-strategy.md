### 🟡 **Tier 2: Memory Persistence & Merging** (Medium Risk)
**Files:** `src/server/routes/memory.ts`, `packages/server/src/routes/v5/dashboard-stubs.ts`

**Changes:**
- **Merge strategy:** Changed from waterfall (fallback) to always-merge. JSONL + KV store + memories.json are now **all merged**, then deduplicated by ID.
- **Filter order:** All filters (search, agent, type, since) applied to full entry set BEFORE 200-entry cap. `meta.total` now reports true match count.
- **Metadata passthrough:** JSONL entries' structured `metadata` field now exposed in API response (previously discarded).

**Test Coverage:** ✅ **Good but incomplete**
- `tests/server/routes/memory.test.ts` — 15 tests covering basic cases
- `packages/server/src/routes/v5/__tests__/memory.test.ts` — 25 tests (JSONL + memories.json merge)
- **Coverage includes:** Empty state, multi-source reads, agent/type filters
- **Coverage gaps:** ⚠️
  - **No explicit test for ID-based dedup collision** (identical id across JSONL + memories.json → only one appears)
  - **No test for metadata field passthrough** when present in JSONL
  - **No test for filter precedence** (does agent filter run BEFORE type filter?)
  - **No test for 200-entry cap after filtering** (do oldest entries get dropped or newest?)

**Risk Factors:**
- Dedup key logic: `const dedupeKey = id || (e.type && e.createdAt ? \`${e.type}:${e.createdAt}\` : undefined)` — fallback syntax-key path untested
- Silent failures on JSON parse (`catch { ... }`) — `_deriveJsonlMeta()` never tested with malformed JSON

**Confidence:** 3.5/5

**Recommended Tests:**
1. `GET /api/v5/memory — dedup by ID removes exact duplicates across JSONL + KV + memories.json`
2. `GET /api/v5/memory?type=gate-verdict — filters BEFORE cap so meta.total reflects true count`
3. `GET /api/v5/memory — metadata field from JSONL entries present in response`

**Risk Factors:**

**Confidence:** 3/5

**Recommended Tests:**
1. `computeMemoryStats — tier-1 hit when audit.json has memoriesInjected > 0`
2. `computeMemoryStats — tier-1 miss when audit.json has memoriesInjected === 0`
3. `computeMemoryStats — failed and running cycles excluded from denominator`

## Missing Test Coverage Concerns (High → Low Priority)

### 🔴 **High Priority**

1. **Memory deduplication by ID (cross-source)**
   - **Why:** New merge strategy (always-merge) not previously tested for duplication edge cases
   - **Test case:** Create identical entry ID in JSONL + memories.json → only one appears in response
   - **File:** `src/server/routes/memory.ts`
   - **Est. complexity:** 1 hour (mock both sources)

2. **Flywheel tier-1 logic (audit.json `memoriesInjected` field)**
   - **Why:** Hit-rate calculation changed to explicit two-tier system; tier-1 logic untested
   - **Test cases:**
     - `memoriesInjected === 1` → hit counts
     - `memoriesInjected === 0` → miss counts
     - Tier-1 field present AND valid → overrides tier-2 proxy
   - **File:** `src/server/routes/flywheel.ts`
   - **Est. complexity:** 2 hours (mock audit.json files, test both paths)

3. **Memory metadata field passthrough**
   - **Why:** Structured metadata from JSONL now exposed in API; no test validates structure
   - **Test case:** `GET /api/v5/memory` returns `data[].metadata` with correct `GateVerdictMetadata` shape
   - **File:** `src/server/routes/memory.ts`
   - **Est. complexity:** 1 hour

### 🟡 **Medium Priority**

4. **Flywheel `stage === 'completed'` denominator filter**
   - **Why:** Running/failed cycles excluded; no explicit test validates this threshold
   - **Test case:** Create completed + failed + running cycles → only completed counts toward denominator
   - **File:** `src/server/routes/flywheel.ts`
   - **Est. complexity:** 1.5 hours

5. **Memory filter order and cap interactions**
   - **Why:** Changed from capping THEN filtering to filtering THEN capping; edge case interactions possible
   - **Test case:** 
     - Apply type + agent filters to 500 entries → should get ≤200 matching, not 200 total then filter
     - `meta.total` should reflect filtered count, not cap count
   - **File:** `src/server/routes/memory.ts`
   - **Est. complexity:** 2 hours

6. **Search term edge cases**
   - **Why:** No validation for special regex chars, very long queries, Unicode
   - **Test cases:** `search("**")`, `search("x" * 1000)`, `search("你好")`
   - **File:** `src/server/routes/search.ts`
   - **Est. complexity:** 1 hour
