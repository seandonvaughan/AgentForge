## Risk Assessment by Changed File

### 🔴 **HIGH RISK** — `packages/core/src/autonomous/cycle-runner.ts` (12 insertions)

**What Changed:**  
PR title sanitization logic replacing simple string interpolation with a complex truncation algorithm:
- Removes parentheses (gh CLI parses `(...)` as option groups when unquoted)
- Collapses newlines to spaces
- Word-boundary truncation at 65 chars max
- Appends ellipsis (`…`) when truncated

**Why High Risk:**
- PR creation is a **core git-ops safety boundary** — titles affect GitHub API calls
- The algorithm has **no unit tests** and is only tested implicitly in cycle-runner end-to-end tests
- Untested edge cases could break or crash the PR opener:
  - Empty title after sanitization (edge case: summary = `(((ignored)))`)
  - Title < 20 chars: `lastSpace > 20` check could fail to find a word boundary
  - Summary with only newlines: `oneLine.length` becomes empty string
  - Version string longer than room (65 - prefix length): results in negative room

**Recommended Tests:**
```
Title sanitization should:
  ✗ remove all parentheses from summary
  ✗ collapse multiple newlines/carriage returns to single space
  ✗ truncate at word boundary preserving prefix + content + ellipsis
  ✗ handle empty/null summary gracefully
  ✗ handle summary > 200 chars (common from AI summaries)
  ✗ reject title < 1 char after sanitization
  ✗ handle version strings up to 10 chars (e.g., v6.10.0)
  ✗ preserve content between first non-space and truncation point
  ✗ don't duplicate ellipsis if summary already ends with …
```

### 🟡 **MEDIUM RISK** — `packages/core/src/autonomous/proposal-to-backlog.ts` (added sanitizeItems, updated regex)

**What Changed:**

1. **Line 203:** Regex group made optional — `(?:(?:\/\/|...|#)[^\n]*?)?` allows plain-text TODO lines in Markdown
2. **Lines 246–257:** New `sanitizeItems()` method strips `-->` and `*/` from every BacklogItem's `id` and `title`
3. **Line 151:** Call chain: `sanitizeItems(deduplicate(items))`

**Why Medium Risk:**
- Output sanitization is **defensive** — catches artifacts from adapter data (session errors, task descriptions)
- New regex is more permissive but guarded by `^` anchor (prevents false positives in const assignments)
- Tests added cover plain-text markers and `-->` stripping
- Deduplication ordering matters: runs *before* sanitization to prevent title collision bugs

**Concerns:**
- Regex change affects **capture groups** — need to verify group indices don't shift
- `sanitizeItems` runs on all items but is only truly necessary for adapter-sourced items (sessions, tasks)
- No test for the interaction: "what if adapter gives item with `-->` that also deduplicates with another item?"

**Recommended Tests:**
```
Regex updates should:
  ✗ capture plain-text TODO in .md files (no <!-- prefix)
  ✗ still reject TODO inside const declaration strings (^-guarded)
  ✗ handle mixed comment styles in same file
  ✗ preserve FIXME priority (P0 vs P1)

Deduplication + sanitization should:
  ✗ deduplicate first, then sanitize (prevents title collision from --> removal)
  ✗ handle item that matches another after sanitization
  ✗ preserve id uniqueness even after --> removal
  ✗ not corrupt title format when both --> and */ present
```
