# Live Activity Feed SSE Verification

## Status: VERIFIED ✅

The `/live` activity feed correctly subscribes to the SSE stream and renders
`cycle_event` messages with proper colors, types, and timestamps.

---

## What Was Verified

1. ✅ The `/live` page subscribes to `/api/v5/stream` via `EventSource` in its `onMount` hook
2. ✅ `cycle_event` messages are received via `onmessage` (unnamed SSE events — wire format is `data: {...}\n\n`)
3. ✅ Events render with status-aware left-border colors (`cycleAccentColor`)
4. ✅ Type badge renders as "Cycle" (from `TYPE_LABELS['cycle_event']`)
5. ✅ Badge styled with `--color-sonnet` CSS variable (defined as `#4a9eff` in `app.css`)
6. ✅ Category tags use `CYCLE_CATEGORY_LABELS` (e.g. `phase.start` → `Phase →`)
7. ✅ Timestamps formatted `HH:MM:SS` (24h, via `formatTime`)
8. ✅ Heartbeat events silently filtered before reaching the feed list
9. ✅ Auto-reconnects after 3 s on connection error
10. ✅ Filter dropdown includes `cycle_event` option

## Color Logic

| Category pattern      | CSS variable          | Resolved color |
|-----------------------|-----------------------|----------------|
| `fail` / `error`      | `--color-danger`      | `#e05a5a`      |
| `complete` / `pass` / `commit` | `--color-success` | `#4caf82` |
| `warn` / `budget`     | `--color-warning`     | `#f5a623`      |
| `start` (default)     | `--color-sonnet`      | `#4a9eff`      |

The type badge always uses `var(--color-sonnet, var(--color-info))` as a graceful
fallback — both variables resolve to `#4a9eff` in the current theme.

## Data Pipeline

```
CycleEventsWatcher (packages/server/src/routes/v5/cycles.ts)
  └─ globalStream.emit({ type: 'cycle_event', category: msg.type, message, data: msg })
       └─ EventStream (packages/server/src/routes/v5/stream.ts)
            └─ reply.raw.write(`data: ${JSON.stringify(event)}\n\n`)
                 └─ EventSource.onmessage in packages/dashboard/src/routes/live/+page.svelte
                      └─ events.update([...list, parsed])
                           └─ Svelte reactive render (.feed-row, .type-badge, .category-tag, .timestamp)
```

All SSE events use the **unnamed** format (`data: …\n\n`) — not named events
(`event: cycle_event\ndata: …\n\n`). This is correct: `EventSource.onmessage`
only fires for unnamed events; named events require `addEventListener`. A
regression to named events would silently drop all `cycle_event` messages from
the feed.

## Event Shape (StreamEvent / FeedEvent)

```typescript
interface StreamEvent {
  id: string;        // generated by generateId() in EventStream.emit
  type: 'cycle_event';
  category: string;  // e.g. 'phase.start', 'cycle.failed'
  message: string;   // e.g. 'abcdef12 · phase.start · execute'
  data?: Record<string, unknown>;  // full CycleEventMessage payload
  timestamp: string; // ISO-8601 from nowIso()
}
```
