## Risk Assessment by Component

### 🔴 HIGH RISK

#### 1. **OAuth2 Auth Plugin Integration** (`src/server/auth/`)
- **Coverage:** 35 unit/integration tests ✓
- **Gap:** No tests for:
  - HTTP OPTIONS/HEAD preflight requests (may bypass auth incorrectly)
  - Custom `excludePaths` with regex patterns
  - Interaction with Fastify hook ordering (preHandler vs onRequest timing)
  - Auth header presence in 404 responses
  - Rate-limit headers carrying auth state
- **Mitigation:** Add 5-8 integration tests covering edge HTTP methods and complex excludePath patterns before production deployment
- **Risk Level:** Medium — core security path; should test against real request types

#### 2. **Query Cache Invalidation Race Conditions** (`src/db/query-cache.ts`)
- **Coverage:** 24 tests covering TTL, tags, SqliteAdapter integration ✓
- **Gap:** No tests for:
  - Concurrent read-while-invalidating scenarios (Promise.all with get/invalidateTag interleaved)
  - Tag cleanup on evict when entry belongs to multiple tags
  - Memory leak if `entries.delete()` is called but tagIndex still references the key
  - Stale cache entry returned if expiry check races with evict
- **Mitigation:** Add stress tests with concurrent operations using async/await interleaving
- **Risk Level:** Medium — Could cause stale session data in autonomous loop if race condition triggers

#### 3. **Search Endpoint** (`packages/server/src/routes/v5/search.ts`)
- **Coverage:** E2E only (no unit tests) ✗
- **Gap:** No validation tests for:
  - Empty/whitespace-only query strings → should return empty array, not crash
  - File system errors (missing `.agentforge/` directory) → should gracefully degrade to adapter-only search
  - Malformed JSON in sprint/cycle files → parser should catch and skip, not crash
  - XSS injection via query terms in results (API should be safe, but verify)
  - Large result sets (limit: 1000) → confirm response time doesn't degrade
  - Query with 100+ terms → tokenizer should cap or paginate
- **Mitigation:** Create `tests/v5/routes/search.test.ts` with 15-20 unit tests covering above scenarios
- **Risk Level:** Medium-High — Missing files could cause 500 errors; malformed data could crash search

### 🟡 MEDIUM RISK

- **Risk Level:** Low-Medium — Core streaming module used in chat/SSE paths; unlikely to fail but edge cases unproven

## Critical Issues & Follow-Up Actions

### Issue #1: **Timeout Failure — DashboardAgent md-12**

**What happened:** DashboardAgent timed out at 600s on sprint item md-12 (2 attempts).  
**Impact:** 1 sprint item unresolved; unclear if task is in a broken loop or legitimately slow.  
**Action Required:**
1. Identify what md-12 task was (check sprint backlog)
2. Run the task locally with profiling to detect:
   - Infinite loop in Playwright test
   - Missing file causing repeated reads
   - O(n²) algorithm on large dataset
3. If a known performance issue, add timeout guard; if a loop, fix it before release

### Issue #2: **Search Endpoint Missing Unit Tests**

**Priority:** HIGH  
**Why:** Unvalidated input handling on file system operations can crash server.  
**Add tests for:**
- Empty query, whitespace-only query → return `{ data: [], agents: [], meta: { total: 0 } }`
- Missing `.agentforge/` directory → gracefully skip file sources
- Malformed JSON in cycle.json → skip that cycle, continue
- Query with 100+ words → tokenizer should cap or dedup efficiently
- Results with limit=0 → should return empty, not error

**Estimated effort:** 4-6 hours; ~20 test cases

### Issue #3: **Cache Race Conditions Untested**

**Priority:** MEDIUM  
**Why:** Autonomous loop hits cache hot-path on every cycle; race condition could leak stale session state.  
**Add tests for:**
- Read while tag invalidation is in flight (interleaved Promises)
- Multiple concurrent writes to same key
- Eviction cleanup when entry has 3+ tags

**Estimated effort:** 3-4 hours; ~8 test cases

### Issue #4: **Auth Plugin HTTP Method Coverage**

**Priority:** MEDIUM  
**Why:** OPTIONS requests (CORS preflight) may bypass auth incorrectly, exposing endpoint surface.  
**Add tests for:**
- OPTIONS /api/v5/* (should 200 without auth, or follow auth rules?)
- HEAD /api/v5/* (should behave like GET)
- Custom excludePaths with glob patterns

**Estimated effort:** 2-3 hours; ~6 test cases
