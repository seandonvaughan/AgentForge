# AgentForge v3.1 Implementation Plan

**Produced:** 2026-03-25
**Authors:** Lead Architect, VP Engineering, VP Product (synthesized)
**Status:** APPROVED — implementation-ready
**Version target:** 0.3.0

---

## Executive Summary

v3.1 closes the gap between the documented product and the running code across 10 identified gaps (M1–M10). The highest-value work is not new architecture — it is wiring what already exists into the product surface. Two sprint tracks run in parallel: the **Platform Track** (Genesis pipeline, brief depth, topology) and the **Runtime Track** (invoke command, control loop, integration dispatch, cost reporting, reforge naming).

---

## Sprint Structure

### Sprint 1 — P0 foundations (both tracks in parallel)

**Platform Sprint 1: M1 + M2 (one PR)**
- Fix `runGenesis()` — replace `forgeTeam()` call with `designTeam()` + `writeTeam()`
- Add `src/genesis/interview-runner.ts` (readline-based, ~60 lines)
- Wire approval gate in `genesisAction()` — `y`/`n` before any file write
- Add `--yes` flag for CI/automation bypass
- Auto-interview when `discoveryState === "empty"`
- Flag rename: `--interview` stays as explicit override; auto-trigger is state-based

**Runtime Sprint 1: M5 + M8 write path (one PR)**
- New `src/cli/commands/invoke.ts` → wires to `AgentForgeSession`
- `session.end()` writes `cost-entry-{sessionId}-{timestamp}.json` to `.agentforge/sessions/`
- `cost-report.ts` reads `cost-entry-*.json` files; no more `cost-data.json`
- benchmark-lead validates with v3 launch gate suite before Sprint 2

### Sprint 2 — P1 (three parallel tracks after Sprint 1 gates)

**Track A — Platform: M3 (brief depth)**
- Wire `scan.documents/integrations/research` through `buildBrief()` (plumbing only — types already correct)
- Extend `inferDomains()` to use `DocumentAnalysis.type` classification signals
- Add `RESEARCH_QUESTIONS` bank to `interviewer.ts` (3 targeted questions: modality, output artifact, data sensitivity)
- Conditional question branching when `project_type === "Research project"`
- `designTeam()` uses research answers to suppress/promote agents (e.g., suppress `ml-engineer` for literature-review modality)
- Owner: genesis-pipeline-dev + system-prompt-crafter

**Track B — Platform: M4 (topology persistence)**
- Populate `manifest.collaboration` in `designTeam()`
- Add `config/topology.yaml` write to `writeTeam()` (one new `writeFile` call)
- Owner: team-composer-dev

**Track C — Runtime: M6 (control loop)**
- New `src/orchestrator/control-loop.ts`
- Loop exits on: max iterations (20), budget exhausted, loop detected, `is_request_satisfied`
- Ships behind `--loop` flag on `invoke` — not default
- Owner: progress-loop-guard-dev; review: event-handoff-context-dev; approval: runtime-platform-lead

**Track D — Runtime: M7 (filesystem integration)**
- Add `"filesystem"` to `IntegrationTarget` union in `src/types/integration.ts`
- Actions: `filesystem:write_file`, `filesystem:read_file` using `fs.promises`
- Real dispatch — no longer returns `{ stub: true }`
- Owner: integration-runtime-dev

**Track E — Runtime: M9 (reforge naming split)**
- New `src/cli/commands/rebuild.ts` — team re-scan (formerly base `reforge`)
- `reforge.ts` keeps only subcommands: apply, list, rollback, status (agent tuning)
- `reforgeTeam()` → `rebuildTeam()` (follow-on rename, not a blocker)
- Owner: delegation-execution-dev; coordination: runtime-platform-lead

### Sprint 3 — M10 (docs/trust alignment, blocks release)

- Bump `package.json` to `0.3.0`
- Update `commands/genesis.md` to reflect M2 behavior (interview UX spec)
- Add "Flags" section to `commands/forge.md`
- Verify and update `commands/reforge.md` (remove unverified `--upgrade` if not wired)
- Split README into "What Works Today" and "Coming in v3.x" sections
- Remove v1/v2/v3 phase history from README → move to `docs/design.md`
- Fix test count (actual: 756 tests, README says 517)
- **5 required E2E tests (block release):**
  1. `genesis` on empty dir — produces valid team.yaml
  2. `genesis --interview` with piped stdin answers — uses answers in brief
  3. `genesis` with `n` on approval — exits 0, `.agentforge/` NOT created
  4. `forge` on TypeScript project — writes team.yaml with Opus + Sonnet routing
  5. `forge --dry-run` — prints plan, does NOT write `.agentforge/`
- Owner: technical-writer (docs) + end-to-end-tester (E2E tests)

## Decisions Reference

### M1 — Genesis Write Path
| Decision | Value |
|---|---|
| Fix | Replace `forgeTeam(projectRoot)` on line 226 of `genesis/index.ts` with `designTeam()` + `writeTeam()` |
| `forgeTeam()` | Untouched — stays as `forge` command entrypoint |
| Risk | Check all tests for `forged_by` assertions before merge (will change from `"agentforge"` to `"genesis"`) |
| Scope | ~10 lines changed in `runGenesis()` |

### M2 — Interview + Approval Gate
| Decision | Value |
|---|---|
| Auto-interview trigger | `discoveryState === "empty"` → force interview |
| Runner | `src/genesis/interview-runner.ts` — Node `readline/promises`, no new deps |
| Approval prompt | `y` accept · `n` cancel. Exit 0 on cancel, nothing written. `e` (edit) deferred to v3.2 |
| CI bypass | `--yes` flag bypasses approval gate |
| Flag convention | `--interview` = explicit opt-in override for any discovery state |
| Team summary | Formatted table: domains, agents grouped by tier with model, role descriptions, cost savings estimate |

### M3 — Brief Depth
| Decision | Value |
|---|---|
| Approach | Plumbing fix in `buildBrief()` — not a new scanner |
| v3.1 non-code type | Research projects only (domain pack already exists) |
| Research questions | 3: modality (choice), output artifact (choice), data sensitivity (text) |
| Branch trigger | When interview `project_type` answer = "Research project" |
| Future types | Data pipeline (v3.2), docs site (v3.3) |

### M4 — Topology Persistence
| Decision | Value |
|---|---|
| Write locations | `team.yaml` (inline `collaboration` field) + `config/topology.yaml` |
| Trigger | `designTeam()` populates; `writeTeam()` writes |

### M5 — v3 Session as Default CLI Path
| Decision | Value |
|---|---|
| Approach | New `src/cli/commands/invoke.ts` → `AgentForgeSession` |
| Legacy | `Orchestrator` stays but is no longer the primary path |

### M6 — Control Loop
| Decision | Value |
|---|---|

| Termination | max_iterations (20) · budget_exhausted · loop_detected · no_progress · completed |

### M7 — Integration Dispatch
| Decision | Value |
|---|---|
| First target | `filesystem` — `fs.promises.readFile/writeFile`, no MCP required |

### M8 — Cost Reporting
| Decision | Value |

### M10 — Docs/Trust
| Decision | Value |
|---|---|

## Risk Register

| Risk | Severity | Mitigation |
|---|---|---|
| `forged_by` assertion failures after M1 | Low | Grep all tests for `forged_by` before merge |
| Non-TTY hang on approval gate | Medium | `--yes` flag is mandatory, document in help text |
| `forge` and `genesis` produce different manifest shapes after M4 | Low | Document — topology in genesis only until v3.2 |
| `--loop` control loop terminates incorrectly | Medium | Ships behind flag; benchmark-lead validates before making default |
| README test count diverges | Low | CI check: fail if count diverges from vitest output by >10% |

## What This Is NOT

- Not a new architecture — all modules exist, this is wiring
- Not v3.2 — hard activation, peer-to-peer agent comms, `$EDITOR` flow are post-v3.1
- Not a 1.0 release — E2E coverage, doc alignment, and stable CLI API come after this cycle
