## Process Observations

### What Worked
1. **Parallel agent execution** — launching 3 agents for independent P0 work cut delivery time 3x
2. **Dogfood sprint model** — real feedback from all agents identified genuine issues
3. **TDD gate methodology** — every phase had pass/fail criteria, no ambiguity
4. **Optional bus param pattern** — backward compatibility preserved perfectly

### What Didn't Work
1. **Dashboard agent hit rate limit** — background agent couldn't complete, required restart
2. **Linter/forge hooks re-ran team.yaml** — forge hook overwrites team.yaml with scanned defaults, losing manual edits
3. **Test count assertion (18 topics)** — hardcoded magic number broke when topics expanded
4. **No CI** — all testing is manual (`npx vitest run`), nothing runs on push

### Recommendations for v4.2+
1. Set up GitHub Actions CI before doing more work
2. Add `.agentforge/team.yaml` to .gitignore for forge hook, or add merge strategy
3. Use `toBeGreaterThanOrEqual` instead of `toBe` for counts that may grow
4. Dashboard should read real data from `.agentforge/` files, not just simulate
