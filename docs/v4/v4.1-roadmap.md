# AgentForge v4.1 Roadmap — Polish & Production Integration

**Synthesized by:** CTO
**Date:** 2026-03-26
**Source:** 17 agent dogfood feedback reports (all agents reported)
**Verdicts:** 15 pass, 2 partial (debugger, researcher — both on semantic search)

---

## Executive Summary

v4.0 delivered 20 modules across 5 pillars with 381 tests and 100% gate pass rate. The dogfood sprint confirmed the architecture is sound and composable. However, **the modules exist in isolation** — they aren't wired into the production runtime (`invoke`, `genesis`, `forge`). v4.1 closes this gap.

The #1 finding across all agents: **semantic search is keyword-only and must be upgraded to embeddings.** The #2 finding: **REFORGE doesn't actually modify files.** The #3 finding: **most modules don't emit bus events.**

## Feedback Synthesis — Themes

### Theme 1: Isolation Problem (mentioned by 8 agents)
Modules work in isolation but aren't connected to each other or the runtime. Bus events, session tracking, and memory storage are all opt-in — nothing is wired by default.

**Fix:** P0 items 1 and 4 — bus unification + runtime integration.

### Theme 2: Search Quality (mentioned by 4 agents)
Bag-of-words cosine similarity is a proof-of-concept, not production. Fails for semantically related but lexically different queries.

**Fix:** P0 item 2 — embedding-based search.

### Theme 3: REFORGE is Simulated (mentioned by 3 agents)
The self-modification engine tracks proposals and guardrails but doesn't actually modify files or run tests.

**Fix:** P0 item 3 — git integration.

### Theme 4: Observability Gap (mentioned by 5 agents)
No logging, no error hierarchy, no health checks, no metrics. Hard to debug in production.

**Fix:** P1 items 5-6 — logging + error classes.

### Theme 5: API Polish (mentioned by 6 agents)
Inconsistent naming, test helpers in production, missing validation, no pagination.

**Fix:** P1-P2 items 7, 9, 10, 18.
