# AgentForge v4.4 Roadmap

**Status:** Draft — pending v4.3 gate approval
**Sprint:** Autonomous v4.4
**Budget:** $450
**Team:** 37 agents (6 Opus, 27 Sonnet, 4 Haiku)
**Focus:** Real intelligence — replace simulations with real agent execution

## Context

v4.3 delivers a production-grade dashboard with 23 improvements. v4.4 shifts focus from
**visualization** to **real execution** — making agents actually invoke Claude, persist results,
and feed live data back into the dashboard built in v4.3.

---

## P0 — Real Agent Execution (must ship)

### P0-1: Runtime Invoke → Real Claude API call
**Owner:** architect + coder
Replace the stub `invoke` command with a real Claude API call using `@anthropic-ai/sdk`.
- Accept `--agent <name>` and `--task <text>` flags
- Load the agent's YAML (system prompt, model, skills)
- Call the correct Claude model (opus/sonnet/haiku per routing)
- Stream the response to stdout
- Record the invocation in `.agentforge/sessions/<id>.json`

### P0-2: Session persistence to disk
**Owner:** coder
`V4Session` must write its state to `.agentforge/sessions/` on every state change.
- `session-<id>.json` with full message history, cost, agent chain
- `sessions/index.json` with summary list (id, agentId, startedAt, status, costUsd)
- Dashboard session timeline reads from `sessions/index.json` (real data)

### P0-3: Bus events from real invocations
**Owner:** coder + meta-architect
Wire `V4MessageBus` to real invoke runs so every tool use and response emits bus events.
- `agent.invoked`, `agent.responded`, `agent.error` on every real call
- `bus/events.json` written by a `BusFileAdapter` for dashboard consumption
- Dashboard Bus Monitor switches from simulation to real events

### P0-4: Fix forge hook team.yaml overwrite
**Owner:** devops-engineer
The `/agentforge:forge` command resets `team.yaml` to the scanned 17-agent roster,
losing manual additions. Fix by:
- Merging scanned agents with existing roster instead of replacing
- Preserving `team_size`, `version`, and custom `delegation_graph` entries
- Adding a `--preserve-manual` flag to forge command

## P1 — Intelligence Improvements

### P1-1: Embedding-based semantic search
**Owner:** rd-lead + coder
Replace TF-IDF in `SemanticSearch` with real embeddings via `text-embedding-3-small`.
- Add `EmbeddingAdapter` interface (real + in-memory for tests)
- Cache embeddings in `.agentforge/embeddings/<hash>.json`
- Similarity search returns ranked results with score > 0.7 threshold
- Benchmark: >98% recall on AgentForge's own codebase search queries

### P1-2: MetaLearningEngine real feedback loop
**Owner:** meta-architect + rd-lead
`MetaLearningEngine` currently classifies patterns but doesn't act on them.
- Read real session outcomes from `.agentforge/sessions/`
- Auto-promote agents that meet promotion criteria (tier 2→3: zero regressions across 5 tasks)
- Write promotion events to `autonomy-tiers.json` (feeds dashboard)
- Alert CEO via bus event when any agent hits tier 4

### P1-3: Multi-agent parallel execution
**Owner:** architect + coder
Allow `invoke` to fan out a task across multiple agents simultaneously.
- `--parallel <agent1,agent2,agent3>` flag
- Each agent runs in its own process (child_process or worker_threads)
- Results merged and compared by `team-reviewer` agent
- Cost and timing reported per-agent

### P1-4: Knowledge ingestion from codebase
**Owner:** rd-lead + researcher
Auto-index the project codebase into the memory system on first run.
- Walk `src/**/*.ts`, parse exports, build semantic index
- Store in `.agentforge/knowledge/codebase.json`
- Agents can query `SemanticSearch` to find relevant code before acting
- Dashboard Memory Browser reads from knowledge index

## P2 — Platform & DX

### P2-1: CLI `agentforge status` command
**Owner:** coder
New command that prints a live dashboard summary to the terminal:
```
AgentForge v4.4  |  37 agents  |  Tier: Autonomous
Sprint 4.4: 8/23 items  |  Budget: $180/$450
Bus: 847 events  |  Sessions: 12 active
```

### P2-2: Plugin auto-update on version change
**Owner:** devops-engineer
When `plugin.json` version changes, automatically:
- Run `npm run build`
- Reload the plugin in Claude Code (via `.claude/settings.json` touch)
- Log the update event to the forge log

### P2-3: Agent YAML validation on startup
**Owner:** linter + qa-automation-engineer
Validate all agent YAML files at startup:
- Required fields: `name`, `model`, `system_prompt`
- Valid model values: `opus`, `sonnet`, `haiku`
- No circular `delegation_graph` references
- Report validation errors as structured JSON, not stack traces

### P2-4: Sprint dashboard reads from real sprint JSON
**Owner:** frontend-dev + dashboard-architect
The Sprint Progress Tracker (v4.3 item 5) currently uses simulated data.
Connect it to `.agentforge/sprints/<version>.json`:
- Load the most recent sprint on page load
- Show real items, assignees, completion status
- 5-second refresh shows live progress during active sprints

### P2-5: Comprehensive API documentation
**Owner:** documentation-writer + api-specialist
Auto-generate API docs from TypeScript source:
- Run `typedoc` over `src/` → `docs/api/`
- Document every exported class, interface, and function
- Add `npm run docs` script
- Link from README
