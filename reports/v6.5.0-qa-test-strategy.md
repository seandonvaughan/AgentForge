## Risk Assessment by Changed File

### `packages/core/src/autonomous/cycle-runner.ts` — **HIGH RISK**

**Change:** PR title generation now sanitizes user-provided summaries (lines 346–360)

#### What Changed
```typescript
// OLD: Simple string interpolation + 50-char slice
title: `autonomous(v${plan.version}): ${scored.summary.slice(0, 50)}`

// NEW: Multi-stage sanitization with edge-case handling
title: (() => {
  const prefix = `autonomous v${plan.version}: `;
  const room = 65 - prefix.length;
  const oneLine = scored.summary
    .replace(/[\r\n]+/g, ' ')        // Collapse multiline
    .replace(/[()]/g, '')             // Remove parens (gh CLI parsing issue)
    .trim();
  if (oneLine.length <= room) return prefix + oneLine;
  const cut = oneLine.slice(0, room);
  const lastSpace = cut.lastIndexOf(' ');
  return prefix + (lastSpace > 20 ? cut.slice(0, lastSpace) : cut) + '…';
})()
```

#### Risk Factors
1. **Complex conditional logic** — Word-boundary truncation at 65 chars with fallback to hard-cut
2. **Multiple regex transformations** — Line collapsing + parenthesis removal interact in subtle ways
3. **gh CLI compatibility** — Parens removal is a workaround for gh CLI argument parsing; incorrect implementation could resurrect the bug
4. **Edge cases unvalidated:**
   - Very short summaries (< 20 chars)
   - Summaries with only parens: `"(fix bug)"` → `"fix bug"` (safe)
   - Summaries with parens at word boundary: `"fix (minor) issue"` → `"fix minor issue"` (truncation point shifts)
   - Multiline summaries with mixed whitespace

## Appendix: Detailed Change Context

**Commit:** `fix(autonomous): PR title sanitization to avoid gh CLI failures`

**Why the change was needed:**
The gh CLI interprets unquoted parentheses as option groups. Example:
```bash
gh pr create --title "autonomous(v6.4.0): fix" # ✓ Quoted, safe
gh pr create --title autonomous(v6.4.0): fix   # ✗ Parens parsed as subshell
```

The sanitization removes parens preemptively, trading a small detail (parens in title) for safety (guaranteed gh CLI compatibility).

**Trade-off:**
- ✅ Prevents gh CLI parsing failures
- ❌ Removes parens from user-facing PR titles (minor cosmetic loss)

**Validation gap:**
The implementation is sound, but the new sanitization logic is untested. This is the QA concern.
